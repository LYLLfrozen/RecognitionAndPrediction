# VFL 实时网络流量监控系统

基于垂直联邦学习(VFL)的实时网络流量监测与识别系统，支持GUI和终端两种运行模式。

## 功能特点

- **垂直联邦学习(VFL)**: 使用隐私保护的联邦学习技术进行流量分类
- **实时监测**: 支持真实网络流量捕获和实时分析
- **混合检测**: 结合机器学习和规则引擎的混合检测策略（ML优先）
- **多种攻击识别**:
  - DoS攻击 (SYN Flood等)
  - 端口扫描 (Probe)
  - 远程登录攻击 (R2L)
  - 提权攻击 (U2R)
  - 正常流量识别
- **GUI界面**: 直观的图形界面，实时可视化流量统计
- **终端模式**: 轻量级终端监控，适合服务器环境

```bash
sudo python3 realtime_monitor.py --interface eth0
```

### 操作说明

1. **选择模式**
   - **真实流量**: 捕获本机网络接口的真实流量（需要管理员权限）
   - **模拟数据**: 使用测试集数据进行演示和准确率验证

2. **开始监测**
   - 点击"开始监测"按钮启动监控
   - 系统自动开始捕获和分析流量
   - 界面每秒自动刷新统计数据

3. **实时统计**
   - **状态**: 显示当前监测状态
   - **总流量包**: 已处理的流量包总数
   - **处理速度**: 每秒处理的流量包数量
   - **运行时间**: 从启动到现在的时间
   - **准确率**: 模型识别准确率（模拟模式）或平均置信度（真实流量模式）

4. **流量分类统计**
   - 进度条显示各类流量的占比
   - 包含5种类别：normal、dos、probe、r2l、u2r

5. **流量可视化**
   - **上图**: 流量处理速率实时曲线，显示最近100秒的数据
   - **下图**: 流量分类分布饼图，直观展示各类流量比例

6. **最近检测记录**
   - 表格显示最近10条检测记录
   - 包含时间、预测类型、置信度、检测方法(ML/rule)

7. **停止监测**
   - 点击"停止监测"按钮停止监控
   - 可以重新开始新的监测任务

## 命令行参数

```bash
python3 realtime_monitor.py [选项]

选项:
  -g, --gui              使用GUI模式（图形界面）
  -s, --sim              使用模拟数据（测试集）而非真实流量
  -i, --interface IFACE  指定网络接口（如: lo0, en0, eth0）
  -d, --duration SECS    运行时长（秒），默认无限运行（仅终端模式）
  -h, --help             显示帮助信息
```

## 检测策略说明

本系统采用**ML优先**的混合检测策略：

1. **主要检测方式**: 机器学习模型(VFL)
   - 使用训练好的VFL模型进行流量分类
   - 支持复杂的攻击模式识别
   - 自适应学习流量特征

2. **规则辅助验证**: 规则引擎
   - 仅在非常明显的攻击特征时覆盖ML结果
   - 提高极端攻击场景的识别率
   - 减少误报率

3. **检测方法标识**:
   - `ml`: 机器学习模型检测（主要）
   - `rule`: 规则引擎检测（辅助）

## 常见问题

### Q1: GUI中文显示乱码怎么办？
**A**: 系统已自动配置中文字体支持，如果仍有问题：
- macOS: 确保系统有 Arial Unicode MS 或 STHeiti 字体
- Windows: 确保系统有 Microsoft YaHei 字体
- Linux: 安装 WenQuanYi 字体 `sudo apt install fonts-wqy-microhei`

### Q2: 启动时提示"需要root权限"？
**A**: 真实流量捕获需要管理员权限：
- macOS/Linux: 使用 `sudo python3 realtime_monitor.py`
- Windows: 以管理员身份运行命令行
- 或者使用模拟模式: `python3 realtime_monitor.py --sim`

### Q3: 提示"scapy未安装"？
**A**: 安装scapy库：
```bash
pip install scapy
```

Windows用户还需要安装 [Npcap](https://npcap.com/)

### Q4: 真实流量模式下没有检测到流量？
**A**: 可能的原因：
1. 网络接口不活跃 - 尝试访问一些网站产生流量
2. 接口选择错误 - 使用 `ifconfig` 查看正确的接口名
3. 权限不足 - 确保使用 `sudo` 运行
4. 防火墙阻止 - 检查防火墙设置

### Q5: 如何生成测试流量？
**A**: 参考项目中的测试工具：
```bash
# 生成各类攻击模拟流量
python3 generate_test_traffic.py
```

### Q6: 为什么大部分流量都被识别为normal？
**A**: 这是正常现象：
- 正常网络环境中，90%以上的流量都是正常流量
- 模型已经过训练，能准确区分正常和异常流量
- 如需测试攻击检测，请使用攻击模拟工具
